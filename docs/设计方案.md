# Claude Code Settings Manager (CSM) 设计方案

## 一、项目背景

### 问题陈述
Claude Code 的 `settings.json` 文件只支持单一配置，当用户需要在不同的模型厂商（如 Anthropic 官方、百度千帆、AWS Bedrock 等）之间切换时，必须手动修改配置文件，操作繁琐且容易出错。

### 解决方案
开发一个 CLI 工具，实现多配置文件共存、一键切换的功能，提升多环境配置管理效率。

---

## 二、核心概念

### 2.1 Profile（配置档案）
每个 Profile 代表一个完整的 Claude Code 配置，包含：
- 名称（唯一标识，通过文件名确定）
- 完整的 settings.json 内容

### 2.2 Active Profile（当前激活配置）
当前生效的配置，通过替换 `~/.claude/settings.json` 实现。

---

## 三、功能目标

### 3.1 配置管理（CRUD）

| 功能 | 命令示例 | 说明 |
|------|----------|------|
| 创建配置 | `csm create <name>` | 创建新的配置文件，支持从当前配置复制 |
| 查看配置 | `csm show <name>` | 显示指定配置的详细内容 |
| 编辑配置 | `csm edit <name>` | 打开编辑器修改配置 |
| 删除配置 | `csm delete <name>` | 删除指定配置文件 |
| 列出配置 | `csm list` / `csm ls` | 显示所有配置档案列表 |
| 重命名配置 | `csm rename <old> <new>` | 重命名配置档案 |

### 3.2 配置切换

| 功能 | 命令示例 | 说明 |
|------|----------|------|
| 切换配置 | `csm use <name>` | 切换到指定配置，无参数时交互式选择 |
| 快速切换 | `csm switch` | 交互式选择 (use 的别名) |
| 查看当前 | `csm current` | 显示当前激活的配置名称 |

### 3.3 配置导入导出

| 功能 | 命令示例 | 说明 |
|------|----------|------|
| 导入配置 | `csm import <file>` | 从外部 JSON 文件导入配置 |
| 导出配置 | `csm export <name>` | 导出指定配置为 JSON 文件 |
| 复制配置 | `csm copy <src> <dest>` | 复制现有配置创建新配置 |

### 3.4 配置备份恢复

| 功能 | 命令示例 | 说明 |
|------|----------|------|
| 备份配置 | `csm backup` | 备份所有配置到压缩包 |
| 恢复配置 | `csm restore <backup>` | 从备份恢复配置 |
| 查看备份 | `csm backups` | 列出所有备份文件 |

## 四、安全设计

### 4.1 Profile 名称验证
- 名称长度限制：最大 64 个字符
- 允许字符：字母、数字、连字符（-）、下划线（_）
- 禁止字符：空格、中文、特殊符号、路径分隔符

### 4.2 路径遍历防护
- 验证所有用户输入的文件路径
- 禁止包含 `..`、`/`、`\` 等危险字符
- 备份恢复时检查 ZIP 条目路径安全性

### 4.3 原子写入机制
- 采用"写入临时文件 + 原子重命名"策略
- 防止写入过程中崩溃导致的数据损坏
- 自动清理残留的临时文件

### 4.4 备份验证
- 恢复前验证备份文件完整性
- 检测 ZIP 条目中的路径遍历攻击
- 报告备份内容（配置数量、文件大小等）

## 六、数据存储设计

### 5.1 目录结构
```
~/.claude/
├── settings.json          # 当前生效配置（由工具管理）
├── csm/                   # CSM 数据目录
│   ├── profiles/          # 配置档案存储目录
│   │   ├── baidu-qianfan.json
│   │   ├── anthropic-official.json
│   │   └── aws-bedrock.json
│   ├── backups/           # 备份存储目录
│   │   └── csm-backup-20260331-120000.zip
│   └── csm-state.json     # 工具状态（当前激活配置等）
```

### 5.2 配置文件格式
每个 profile 的 JSON 文件直接存储 ClaudeSettings 内容，文件名即为配置名称：
```json
{
  "env": { ... },
  "permissions": { ... },
  "enabledPlugins": { ... },
  ...
}
```
说明：不包含额外的 name、description、时间戳等元数据。

### 5.3 状态文件格式
`csm/csm-state.json` 记录当前状态：
```json
{
  "activeProfile": "baidu-qianfan",
  "version": "1.0.0"
}
```

## 九、配置合并策略

切换配置时支持部分字段继承，设计如下：

### 9.1 合并规则
```bash
csm use <name> [options]

选项:
  --merge <fields>    指定要合并的字段（逗号分隔）
  --no-merge          完全替换，不合并任何字段
  --keep-permissions  保留当前权限配置
  --keep-plugins      保留当前插件配置
```

### 9.2 合并示例
```bash
# 完全替换（默认行为）
csm use anthropic-official

# 切换模型但保留权限和插件配置
csm use openai-proxy --keep-permissions --keep-plugins

# 只合并特定字段
csm use baidu-qianfan --merge permissions,enabledPlugins
```

### 9.3 合并优先级
1. 目标配置（要切换到的配置）为基础
2. 当前配置中被指定的合并字段覆盖目标配置的对应字段
3. 合并结果写入 settings.json
